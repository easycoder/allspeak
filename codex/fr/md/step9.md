L'étape précédente dessinait un seul rectangle dont la hauteur suivait une courbe sinusoïdale. Mets-en plusieurs ensemble et fais-les démarrer chacun un peu après le précédent, et tu obtiens une Ola (la « Mexican Wave »). Le code est un peu complexe et je ne te demande pas de le comprendre entièrement, mais j'aimerais que tu voies comment il utilise les tableaux que j'ai introduits il y a quelques étapes. Comme précédemment, le mieux est de copier le code dans l'éditeur puis de l'exécuter, pour voir de quoi il a l'air.

Certaines parties du code ressemblent à l'étape précédente, mais la différence principale est qu'il y a 10 rectangles, et non un seul. Ils sont placés côte à côte dans le conteneur ; encore une fois le CSS est délicat à expliquer, mais chaque partie a son utilité. Les rectangles doivent utiliser un positionnement absolu, ce qui signifie qu'ils ont des positions fixes au lieu d'être simplement disposés à travers la page comme le navigateur le juge bon. Cela poserait problème car lorsqu'un rectangle bouge, il entraînerait les autres avec lui. Les composants ne peuvent avoir un positionnement absolu que si leur parent (le conteneur) a un positionnement relatif ; ne me demande pas pourquoi — c'est comme ça.

Parlons maintenant de ces tableaux. Une variable n'est qu'un conteneur pour une seule valeur, mais parfois on aimerait gérer plusieurs valeurs ensemble dans un groupe, comme pour les 10 rectangles. On pourrait simplement créer ~code:Rectangle1~, ~code:Rectangle2~ et ainsi de suite, mais alors tout le code qui les gère devrait aussi être dupliqué. Mieux vaut les avoir comme une seule variable avec 10 éléments et sélectionner chacun à son tour pour travailler avec. Utiliser des tableaux réduit considérablement la quantité de code nécessaire, et moins il y a de code, moins il y a d'erreurs. Les rectangles se comportent tous de façon identique donc il est facile de mettre en place une boucle pour les construire, en utilisant des calculs pour définir les différences de position.

Lorsque le script s'exécute, il maintient un compteur appelé ~code:Time~ qui ne cesse d'augmenter. Chaque rectangle a une valeur ~code:Start~ qui indique quand il doit commencer à jouer son rôle dans l'Ola ; il a aussi son propre ~code:Angle~ privé et une variable appelée ~code:Done~ qui vaut vrai ou faux selon que le cycle est terminé pour ce rectangle. Cette dernière, on l'appelle variable _booléenne_ ou _drapeau_. À chaque étape de ~code:Time~, le script configure tous les tableaux pour pointer sur le premier rectangle et vérifie si l'heure de départ a été atteinte. Si oui, il calcule la nouvelle position et taille du rectangle et le déplace. Puis il répète la procédure pour les autres rectangles. Une fois que le cycle est terminé pour un rectangle, le drapeau ~code:Done~ est activé et le test à la ligne 69 garantira qu'aucune autre action ne sera prise pour ce rectangle. Une fois qu'ils ont tous fini, tout le processus recommence depuis le début. Dans l'exemple précédent j'ai utilisé une boucle dans une boucle, mais ici j'ai utilisé une étiquette et un ~code:goto~ explicite. Beaucoup de programmeurs professionnels font semblant d'être horrifiés par cette pratique mais je leur dis « Ne soyez pas si précieux — si c'est clairement compréhensible et que ça fonctionne, c'est bien ».

Comme je l'ai dit plus haut, c'est un exemple relativement complexe, choisi pour mettre en évidence l'utilisation des tableaux mais aussi pour démontrer le genre de choses pour lesquelles ~ec~ peut être utilisé. Aussi difficile que cela puisse paraître, essayer de le faire dans n'importe quel autre langage informatique serait beaucoup plus ardu et le programme correspondant serait bien, bien plus long. La majeure partie du travail ici est faite par le navigateur ; tout ce que nous faisons, c'est lui injecter des valeurs.
